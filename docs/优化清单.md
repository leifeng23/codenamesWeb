# Cosmere CodeNames 优化清单

> 三界宙版 Codenames（群友体验版）代码审查与优化方案。
> 生成日期：2026-06-14。审查范围：`packages/shared`（规则）、`apps/realtime`（实时服务）、`apps/web`（前端 + API）、`prisma`（数据模型）。

实施顺序建议：**规则/逻辑修复 → 夜空背景 + 翻牌动画 → 三类结果特效 + 音效 → 胜负结算 → 体验细节 → 工程/性能**。

---

## 一、规则 / 逻辑问题（影响正确性，优先级最高）

### 1. 没有胜负判定与展示 ⭐ 最高优先
`Game` 模型无 winner 字段，`revealCard` 结束时只把 `phase` 置为 `finished`，前端只显示「游戏结束」。

- 翻完本队最后一张牌获胜，玩家看不到「谁赢了」。
- 踩到刺客（炸弹）应判**翻牌方立即负、对方胜**，现在既不记录也不提示。

**方案**：胜负可由现有状态推导，无需数据库迁移——
- `redRemaining === 0` → 红胜；`blueRemaining === 0` → 蓝胜；
- 其余 finished（踩刺客）→ 输家 = 当前回合队（翻刺客者），赢家为另一队。

在 `buildRoomSnapshot` 中计算 `winnerTeam` 并在前端展示结算画面。

### 2. 猜对 / 猜错 / 炸弹没有区分反馈（特效前置条件）⭐
`card:revealed` 事件只判断「是不是刺客」，前端甚至无法可靠判断本次是猜对还是猜错（快照到达时 `turnTeam` 可能已切换）。

**方案**：服务端在事件里下发结果分类 `outcome: 'correct' | 'neutral' | 'enemy' | 'assassin'` + 猜测队伍 + `winnerTeam`，作为差异化特效与音效的数据来源。

### 3. 线索数量输入未做客户端钳制
`clueCount` 用 `Number(e.target.value)`，清空时变成 `NaN/0`，仅靠服务端 zod 拦截，用户得到笼统报错。`min/max` 仅在标签上不生效。

### 4. 线索词未校验是否为台面词
标准规则禁止用棋盘上的词作线索，当前未拦截（朋友局可选做）。

## 二、操作体验问题

### 5. 「重开一局」无二次确认
已有对局时「开始游戏」会变「重开一局」，点一下立刻重置棋盘，易误触。

### 6. 开局权限偏宽
服务端允许「房主或任意带间谍权限者」开局/重开，对方队间谍也能中途重置，建议收紧为房主或加确认。

### 7. 当前线索对猜牌者不够突出
线索只在右侧小字展示，群体围观应做顶部醒目横幅。

### 8. 错误提示不会自动消失，点击噪音大
`setError` 一直挂着；非当前队点任意牌都会先播放点击音再弹错误。

### 9. 缺少操作日志 / 回看
首页文案承诺「方便回看」，但 `GameEvent` 全存却未在房间展示。

### 10. 移动端棋盘需横向滚动
5×5 棋盘 `min-width:560px`，手机群友需左右拖，建议自适应缩放。

### 11. 缺少回合计时器、加载/禁用态
提交线索、翻牌按钮无 pending 状态；计时器有助于控制群体节奏（可选）。

## 三、代码结构 / 性能

### 12. `game-state.ts` 双份拷贝
`apps/web/lib/game-state.ts` 与 `apps/realtime/src/game-state.ts` 几乎相同且已细微分叉；`apps/web/app/api/games/[id]/*` 这些 REST 路由前端未使用（UI 全走 socket），属易腐化死路径。建议核心逻辑抽到 `@cosmere/shared` 统一维护。

### 13. 每次快照都重算整棵题库树
`buildRoomSnapshot` 每次调用 `buildCategoryTree()`（两次查询），`emitSnapshot` 又为每个 socket 单独构建快照。N 个玩家每操作一次 = N 份完整快照 + N 次题库树查询。建议缓存题库树或拆分推送。

## 四、UI 夜空风格 & 特效方案

现状：背景 `#070812` + 两个径向渐变 + three.js 星点（300 点缓慢旋转）；卡牌扁平色块，翻牌仅瞬间换色；音效为 4 个合成单音。`framer-motion` + `three` 已就绪，特效无需新增重依赖。

### 夜空背景
多层视差星空 + 星星闪烁（明暗呼吸）+ 偶发流星；更深的星云分层渐变（靛蓝/紫/青）+ 暗角 + 缓慢漂移星雾。

### 卡牌质感
玻璃拟态 + 内发光；未翻开的牌做轻微呼吸微光、边框流光。

### 翻牌特效
`rotateY` 真正 3D 翻转（正面词、背面阵营色 + 图标 + 光晕），替换瞬切换色；翻开瞬间加粒子迸发。

### 猜对 / 猜错 / 炸弹特效
- 猜对：本队色光环扩散 + 卡牌脉冲 + 上扬提示音。
- 猜错（中立）：灰色尘埃消散、卡牌变暗、闷响。
- 猜错（对方色）：翻成对方色 + 屏幕边缘对应色闪光（回合结束）。
- 炸弹/刺客：屏幕震动 + 暗红冲击波全屏覆盖 + 低频爆鸣 + 暗角闪烁，接判负结算。

### 胜负结算
获胜方颜色星爆/彩带 + 胜利音效结算横幅；刺客判负明确提示。

### 音效
现仅 click/reveal/assassin/start，需补：猜对、猜错-中立、猜错-对方、胜利、失败。

> ⚠️ 关键依赖：第 2 条（服务端下发 outcome 分类）必须先做，猜对/猜错/炸弹的差异化特效与音效才有数据来源。
